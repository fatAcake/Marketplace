# Bliss

# Мини-маркетплейс

Веб-приложение для размещения товаров пользователями, просмотра каталога и оформления заказов. Платформа позволяет пользователям выступать как в роли покупателей, так и продавцов.

##  Возможности проекта

### Для всех пользователей
-  Просмотр каталога товаров
-  Детальная информация о товаре
-  Просмотр информации о других пользователях
-  Просмотр товаров конкретного продавца

### Для авторизованных пользователей (Покупатель)
-  Оформление заказов с несколькими товарами
-  Просмотр истории своих заказов
-  Редактирование профиля
-  Просмотр аналитики по ценам и скидкам

### Для авторизованных пользователей (Продавец)
-  Создание товаров
-  Редактирование собственных товаров
-  Удаление собственных товаров
-  Управление скидками на товары
-  Аналитика продаж по своим товарам
-  История изменений цен и скидок

### Бизнес-возможности
-  Система скидок (процентные скидки на товары)
-  Аналитика продаж и ценообразования
-  Загрузка изображений товаров
-  JWT аутентификация с refresh-токенами
-  Подтверждение действий через email (опционально)
-  Автоматическое отслеживание истории изменений цен

##  Use Case сценарии

### UC-1: Регистрация пользователя
**Актор:** Гость
**Предусловие:** Пользователь не авторизован
**Основной поток:**
1. Пользователь переходит на страницу регистрации
2. Вводит никнейм, email и пароль
3. Отправляет форму
4. Вводит код из письма для регистрации
5. Система создаёт нового пользователя со статусом "Buyer"
6. Пользователь перенаправляется на страницу входа

**Постусловие:** Пользователь зарегистрирован

---

### UC-2: Авторизация
**Актор:** Зарегистрированный пользователь
**Предусловие:** Пользователь имеет учётную запись
**Основной поток:**
1. Пользователь вводит email и пароль
2. Система проверяет учётные данные
3. Система выдаёт JWT access и refresh токены
4. Пользователь перенаправляется на главную страницу

**Альтернативный поток:**
- При неверных данных система возвращает ошибку

---

### UC-3: Создание товара
**Актор:** Продавец (авторизованный пользователь)
**Предусловие:** Пользователь авторизован
**Основной поток:**
1. Пользователь переходит на страницу создания товара
2. Вводит название, цену, количество, описание
3. Опционально загружает изображения
4. Отправляет форму
5. Система создаёт товар и связывает с владельцем
6. Система фиксирует начальную цену в истории

**Постусловие:** Товар создан и отображается в каталоге

---

### UC-4: Оформление заказа
**Актор:** Покупатель (авторизованный пользователь)
**Предусловие:** Пользователь авторизован, корзина не пуста
**Основной поток:**
1. Пользователь добавляет товары в корзину
2. Переходит к оформлению заказа
3. Система проверяет доступность товаров
4. Система проверяет, что покупатель не покупает свой товар
5. Система рассчитывает итоговую сумму с учётом скидок
6. Система уменьшает количество товаров на складе
7. Создаётся заказ со статусом "Pending"
8. Создаются элементы заказа (OrderItem)

**Альтернативный поток:**
- Если товара недостаточно или покупатель пытается купить свой товар — заказ не создаётся

**Постусловие:** Заказ создан, товары зарезервированы

---

### UC-5: Управление скидками
**Актор:** Продавец
**Предусловие:** Пользователь авторизован, является владельцем товара
**Основной поток:**
1. Продавец выбирает свой товар
2. Создаёт или редактирует скидку
3. Указывает размер скидки и период действия
4. Система сохраняет скидку
5. Система фиксирует изменение в истории цен

**Постусловие:** Скидка применена к товару

---

### UC-6: Просмотр аналитики
**Актор:** Продавец / Администратор
**Предусловие:** Пользователь авторизован
**Основной поток:**
1. Пользователь переходит в раздел аналитики
2. Выбирает период времени
3. Система возвращает данные:
   - Аналитика продаж (количество, сумма)
   - Аналитика цен и скидок
4. Отображение в виде графиков

---

### UC-7: Редактирование товара
**Актор:** Продавец (владелец товара)
**Предусловие:** Пользователь авторизован, является владельцем товара
**Основной поток:**
1. Продавец открывает редактирование своего товара
2. Изменяет поля (название, цена, количество, описание)
3. Отправляет форму
4. Система обновляет товар
5. Система фиксирует изменение цены в истории

**Альтернативный поток:**
- Попытка редактировать чужой товар блокируется

---

### UC-8: Просмотр истории заказов
**Актор:** Покупатель
**Предусловие:** Пользователь авторизован
**Основной поток:**
1. Пользователь переходит в раздел заказов
2. Система отображает список заказов пользователя
3. Пользователь может просмотреть детали каждого заказа

## Технологический стек

### Backend
- **Платформа:** ASP.NET Core
- **Язык:** C#
- **База данных:** PostgreSQL
- **ORM:** Entity Framework Core
- **Аутентификация:** JWT (Access + Refresh токены)
- **Хранение файлов:** BYTEA в PostgreSQL

### Frontend
- **Фреймворк:** React 19
- **Сборщик:** Vite
- **Роутинг:** React Router DOM v7
- **HTTP клиент:** Axios
- **Стили:** CSS

### DevOps
- **Контейнеризация:** Docker
- **Контроль версий:** Git

##  Запуск для разработки

### Предварительные требования

- **.NET SDK 9.0+** ([скачать](https://dotnet.microsoft.com/download))
- **Node.js 18+** ([скачать](https://nodejs.org/))
- **PostgreSQL 14+** ([скачать](https://www.postgresql.org/download/))
- **Git**

### Шаг 1: Клонирование репозитория

```bash
cd /workspace
git clone [<repository-url>](https://github.com/fatAcake/Bliss_marketplace)
cd Bliss_marketplace
```

### Шаг 2: Настройка базы данных PostgreSQL

1. Установите PostgreSQL (если не установлен)
2. Создайте базу данных:

```sql
CREATE DATABASE Marketplace;
```

3. Создайте пользователя (опционально):

```sql
CREATE USER postgres WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE Marketplace TO postgres;
```

### Шаг 3: Настройка Backend

1. Перейдите в директорию backend:

```bash
cd backend/backend
```

2. Создайте файл `.env` на основе примера:

```bash
cp ENV_example.txt .env
```

3. Отредактируйте `.env` файл:

```env
# JWT Settings
JWT_SECRET_KEY=your_secret_32_characters_min
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=7

# Database
CONNECTION_STRING=Host=localhost;Port=5432;Database=Marketplace;Username=postgres;Password=your_password

```

4. Восстановите зависимости и запустите миграции:

```bash
dotnet restore
dotnet ef database update
```

5. Запустите backend:

```bash
dotnet run
```

Backend будет доступен по адресу: `http://localhost:5035`

### Шаг 4: Настройка Frontend

1. Откройте новый терминал и перейдите в директорию frontend:

```bash
cd frontend
```

2. Установите зависимости:

```bash
npm install
```

3. Запустите development сервер:

```bash
npm run dev
```

Frontend будет доступен по адресу: `http://localhost:5173`

### Шаг 5: Проверка работы

1. Откройте браузер и перейдите на `http://localhost:5173`
2. Зарегистрируйте нового пользователя
3. Войдите в систему
4. Создайте товар или оформите заказ

##  Конфигурация

### Переменные окружения Backend

| Переменная | Описание | Пример |
|------------|----------|--------|
| `JWT_SECRET_KEY` | Секретный ключ для JWT (мин. 32 символа) | `you_secret_32_characters` |
| `JWT_ALGORITHM` | Алгоритм подписи JWT | `HS256` |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Время жизни access токена (минуты) | `30` |
| `REFRESH_TOKEN_EXPIRE_DAYS` | Время жизни refresh токена (дни) | `7` |
| `SMTP_EMAIL` | Email для отправки уведомлений | `your_email@gmail.com` |
| `SMTP_PASSWORD` | Пароль приложения SMTP | `your_app_password` |
| `SMTP_SERVER` | SMTP сервер | `smtp.gmail.com` |
| `SMTP_PORT` | SMTP порт | `587` |
| `CONNECTION_STRING` | Строка подключения к PostgreSQL | `Host=localhost;...` |

### Основные группы эндпоинтов:
####  Authentication & Users
- `POST /api/auth/register` — Регистрация
- `POST /api/auth/login` — Вход
- `GET /api/users/me` — Информация о текущем пользователе
- `PUT /api/users/me` — Редактирование профиля

####  Products
- `GET /api/products` — Список товаров
- `GET /api/products/:id` — Детали товара
- `POST /api/products` — Создать товар
- `PUT /api/products/:id` — Редактировать товар
- `DELETE /api/products/:id` — Удалить товар
- `GET /api/products/{id}/images`  — Картинки продукта      
- `GET /api/products/{id}/images/{imageId}`  — Получить картинку       
- `POST /api/products/{id}/images`  — Добавить картинку     
- `DELETE /api/products/{id}/images/{imageId}` — Удалить картинку        
- `PUT/api/products/{id}/images/reorder`  — Изменить порядок картинок

####  Orders
- `POST /api/orders` — Создать заказ
- `GET /api/orders` — Мои заказы
- `GET /api/orders/:id` — Детали заказа
- `PUT /api/orders/:id/status` — Изменить статус

####  Discounts
- `POST /api/discounts` — Создать скидку
- `GET /api/discounts` — Все скидки
- `PUT /api/discounts/:id` — Обновить скидку
- `DELETE /api/discounts/:id` — Удалить скидку

####  Analytics
- `GET /api/analytics/sales` — Аналитика продаж
- `GET /api/analytics/prices` — Аналитика цен
- `GET /api/products/:id/price-history` — История цен товара
