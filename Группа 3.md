# ФУНКЦИОНАЛЬНОЕ ТЗ

## Проект: Мини-маркетплейс

---

# 1. 🎯 Назначение системы

Система предназначена для:

- размещения товаров пользователями
- просмотра товаров
- оформления заказов

Пользователи взаимодействуют друг с другом через покупку и продажу товаров.

---

# 2. 👤 Пользователи и роли

## 2.1 Общие возможности

Система должна позволять:

- регистрировать пользователей
- хранить список пользователей
- просматривать информацию о пользователе

---

## 2.2 Роли пользователей

### 🛍 Покупатель

- просматривает товары
- оформляет заказы

### 🏪 Продавец

- размещает товары
- управляет своими товарами

---

### Важно

- один пользователь может одновременно быть покупателем и продавцом
- отдельного выбора роли при регистрации нет
- роль определяется действиями пользователя

---

# 3. 📦 Товары

## 3.1 Управление товарами (продавец)

Продавец может:

- создавать товар
- редактировать только свои товары
- удалять только свои товары
- просматривать список своих товаров
- просматривать детали своих товаров

---

## 3.2 Просмотр товаров (все пользователи)

Любой пользователь может:

- просматривать список всех товаров
- просматривать детальную информацию о товаре

---

## 3.3 Требования к товару

- название обязательно
- цена > 0
- количество ≥ 0
- у товара есть владелец

---

# 4. 🛒 Заказы

- пользователь может создать заказ
- заказ может содержать один или несколько товаров
- указывается количество каждого товара
- пользователь может просматривать свои заказы и детали каждого заказа

---

# 5. ⚙️ Бизнес-правила заказов

- система проверяет доступность товара
- пользователь не может купить свой товар
- если товара недостаточно - заказ не создаётся
- если заказ допустим - количество товара на складе уменьшается


---

# 6. 🔄 Статусы заказа

- Создан
- Завершён

> Опционально можно использовать “Отменён” или другие статусы для модулей.

---

# 7. 🚫 Ограничения

- нельзя редактировать или удалять чужие товары
- нельзя купить собственный товар
- нельзя создавать товары с некорректными данными

---

# 8. 💾 Хранение данных

- данные должны сохраняться между перезапусками системы

---

# 9. ❗ Обработка ошибок

Система должна:

- предотвращать некорректные действия
- корректно обрабатывать ввод
- не допускать неконсистентного состояния данных

# 10. **Скидки** и ценообразование

- товар может иметь скидку
- Скидку задает продавец
- цена в заказе отображается с учётом скидки
- Аналитика по ценообразованию. Фиксируются все изменения цены и скидки. Пользователь может выбрать период и посмотреть аналитику.

# 11. 📊 Ожидаемый результат

Система должна:

- реализовывать все базовые сценарии
- корректно работать с пользователями, товарами и заказами
- учитывать ограничения
- опциональные модули и модификаторы могут быть добавлены для дифференциации команд
- корректно обрабатывать ошибки

---

# 12. ⚠️ Примечания

- Все решения по архитектуре, технологиям, структуре кода - **определяет команда**
- Базовый функционал должен работать **без модулей и модификаторов**
- Модули и модификаторы - надстройка для усложнения и расширения функционала
